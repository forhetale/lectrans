# LecTrans - 产品需求文档 (PRD)

## 1. 产品概述

### 1.1 产品名称
**LecTrans** (Lecture Translator)

### 1.2 产品定位
> "无感、精准、实时的学术助听器" —— 专为在韩国留学的中国学生设计的课堂实时翻译工具。

### 1.3 目标用户
| 用户画像 | 特征 | 核心痛点 |
|----------|------|----------|
| 韩国留学生 | 中文母语，韩语水平初中级 | 课堂听不懂，笔记记不全 |
| 计算机工学学生 | 有技术背景，愿意折腾 | 专业术语翻译不准 |
| 课堂场景 | 不便外放声音，需要低调使用 | 普通翻译软件太显眼 |

### 1.4 产品价值
- **实时性**：3-5 秒延迟，跟上课堂节奏
- **隐蔽性**：暗黑模式 + 静音设计，不打扰他人
- **智能性**：LLM 驱动，专业术语准确
- **实用性**：一键总结，自动生成笔记

---

## 2. 功能需求

### 2.1 核心功能（MVP）

#### F1: 实时语音转录
| 项目 | 描述 |
|------|------|
| 功能描述 | 实时采集麦克风音频，转录为韩语文本 |
| 输入 | 麦克风音频流 |
| 输出 | 韩语文本（流式） |
| 性能要求 | 延迟 < 3 秒，准确率 > 90% |
| 技术方案 | PyAudio + WebRTC VAD + Groq Whisper API |

#### F2: 实时韩中翻译
| 项目 | 描述 |
|------|------|
| 功能描述 | 将韩语实时翻译为中文，流式输出 |
| 输入 | 韩语文本 |
| 输出 | 中文翻译（流式） |
| 性能要求 | 延迟 < 2 秒 |
| 技术方案 | DeepSeek API / OpenAI 兼容 API |

#### F3: 一键智能总结
| 项目 | 描述 |
|------|------|
| 功能描述 | 将已转录内容发送给 LLM，生成结构化笔记 |
| 输入 | 历史转录文本 |
| 输出 | 结构化 Markdown（大纲、要点、作业/考试） |
| 触发方式 | 点击"总结"按钮 |
| 技术方案 | LLM API + 专用 Prompt |

#### F4: 课堂笔记存档
| 项目 | 描述 |
|------|------|
| 功能描述 | 讲座结束后保存原文、译文和总结 |
| 输入 | 转录记录 + 翻译记录 + 总结 |
| 输出 | Markdown 文件 / PDF |
| 存储位置 | 本地文件系统 |

### 2.2 辅助功能

#### F5: API 配置管理
- 支持自定义 base_url 和 api_key
- 支持多种 LLM 提供商（OpenAI、DeepSeek、Groq、LM Studio）
- 连接状态检测

#### F6: 界面设置
- 暗黑模式（默认）
- 字体大小调节
- 语言对切换（预留）

## 3. 非功能需求

### 3.1 性能需求
| 指标 | 目标值 |
|------|--------|
| 语音转录延迟 | < 3 秒 |
| 翻译延迟 | < 2 秒 |
| 总结生成时间 | < 10 秒 |
| 内存占用 | < 500MB |
| CPU 占用 | < 30% |

### 3.2 可用性需求
- 单文件运行，无需复杂安装
- 支持 Windows/macOS/Linux
- 界面简洁，3 个按钮即可操作

### 3.3 兼容性需求
- Python 3.9+
- 支持主流浏览器（Web 版）

## 4. 用户故事

### 4.1 核心用户故事
```
作为一名在韩国学计算机的留学生，
我希望在课堂上能实时看到教授讲话的中文翻译，
以便跟上课程进度，做好笔记。
```

### 4.2 详细用户故事
| ID | 用户故事 | 验收标准 |
|----|----------|----------|
| US-01 | 我可以一键开始录音和翻译 | 点击 Start 按钮后 3 秒内开始显示翻译 |

## 5. 技术约束

### 5.1 技术栈
| 层次 | 技术选型 | 理由 |
|------|----------|------|
| 音频采集 | PyAudio | Python 原生支持，跨平台 |
| 语音识别 | Groq Whisper API | 速度快，免费额度 |
| 翻译 | DeepSeek API | 性价比高，韩中质量好 |
| 前端 | Streamlit | 快速原型，Python 全栈 |
| 后端 | FastAPI（可选） | WebSocket 支持 |

### 5.2 API 依赖
| API | 用途 | 费用 |
|-----|------|------|
| Groq Whisper | 语音转录 | 免费额度 |
| DeepSeek | 翻译 + 总结 | 约 ¥0.1/1K tokens |

## 7. 风险与缓解

| 风险 | 影响 | 缓解措施 |
|------|------|----------|
| 课堂噪音影响识别 | 高 | 加入 WebRTC VAD 降噪 |
| API 延迟过高 | 中 | 支持本地模型（LM Studio） |
| 专业术语不准 | 中 | Prompt 中加入专业背景约束 |
| 网络不稳定 | 中 | 支持离线缓存，网络恢复后补翻 |

## 附录: Prompt 设计

### 翻译 Prompt
```
你是一位资深的韩中翻译官，专精于学术翻译。
请将以下韩语课堂内容翻译成自然流畅的中文。

要求：
1. 保持双语对照格式（韩语原文 | 中文翻译）
2. 计算机专业术语使用标准中文译法
3. 外来语保留原文并标注（如：알고리즘 - Algorithm）
4. 保持口语化风格，不要过度书面化

韩语原文：
{text}
```

### 总结 Prompt
```
你是一位学术笔记整理专家。
请根据以下韩语课堂转录内容，整理成结构化的中文笔记。

输出格式：
## 📚 核心概念
1. [概念名称]：[简要解释]
2. ...

## 📝 重要知识点
- [知识点1]
- [知识点2]

## 📌 作业/考试信息
- [如有提及，单独列出]

## ❓ 待确认问题
- [如有不确定的内容，列出供课后确认]

转录内容：
{transcript}
```
